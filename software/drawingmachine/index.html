Custom image rendering software
C++, OpenFrameworks, OpenGL
Details
The software tries to represent an image by arranging shapes in space. A stochastic process tries random changes to set of shapes arranged in space. A camera views the arrangement from a single vantage point, and determines if the image resulting from the random change is closer to the target photographic image. If yes, this arrangement is adopted as the new base from which random changes are attempted. Creating a rendering of a photograph involves many iterations where the computer tries random changes. Millions or Billions of changes are rejected and the drawing is reverted to the best-yet version. Once in a while one of the random changes results in an improvement and is accepted into the drawing. This process models evolution in nature.
Time-lapse footage of the drawing converging on a target image.
Marks in the drawing are created by deforming 2D grids of lines in 3D space with genetic programs. More on this method of deformation can be found here. Shapes are then arranged randomly to configure the drawing. There's an inherent tension between the genetic program deformations and the overall constrained goal of a realistic rendering. Genetic programs can be configured and evolved to create wildly expansive shapes and patterns, which are also then being selected for how well they conforom to the goals of the community as a whole.
