The stochastic process
The software builds a representation of a photograph by arranging groups of shapes.

This involves a stochastic process which tries random changes to the arrangement. We capture an image of the arrangement and compare it with the target image we're trying to render. If the random changes have created a better image than what we had before, then the new changes are retained. If the changes made the rendering worse then they are thrown out. Almost every time, the changes make the rendering worse. This is like mutation in biological organisms. A mutation usually damages the organism and makes it worse at staying alive. But occassionally a mutation produces an advantage for the animal. The change is then retained when the animal reproduces.

Just like in nature, creating a rendering of a photograph with this process involves many generations where the computer tries random changes. Billions of attempted changes are rejected and only rarely does a better rendering appear. Of course, unlike nature, the computer can do this very quickly.
Genetic programs
Marks in the drawing are created by deforming 2D grids of lines in 3D space with genetic programs. More on this method of deformation can be found here. Shapes are then arranged randomly to configure the drawing. There's an inherent tension between the genetic program deformations and the overall constrained goal of a realistic rendering. Genetic programs can be configured and evolved to create wildly expansive shapes and patterns, which are also then being selected for how well they conforom to the goals of the community as a whole.
