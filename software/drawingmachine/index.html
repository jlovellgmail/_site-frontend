The stochastic process
The software builds a representation of a photograph by arranging groups of shapes.

This involves a stochastic process which tries random changes to the arrangement. We capture an image of the arrangement and compare it with the target image we're trying to render. If the random changes have created a better rendering than what we had before, then the new changes are retained. If the changes made the rendering worse then they are thrown out. Almost every time, the changes make the rendering worse. This is like mutation in biological organisms. A mutation usually damages the organism and makes it worse at staying alive. But occassionally a mutation produces an advantage for the animal. The change is then retained when the animal reproduces.

Just like in nature, creating a rendering of a photograph with this process involves many generations where the computer tries random changes. Billions of attempts are rejected and only rarely does a better rendering appear. Of course, unlike nature, the computer can do this very quickly.
Genetic programs
Each individual shape in the group is deformed by functions that resemble genetic programs. These functions are represented as trees, and mutation involves breaking off and regenerating branches of the tree. The shapes themselves start out as lines arranged regularly in a 2D plane, like the lines on graph paper. The genetic programs take the original coordinates of these lines as input, and their output is to move the line somewhere else in three dimensional space. More on this method of deformation can be found here. The deformed shapes are then arranged randomly to create the rendering.

There's an inherent tension between the genetic program deformations and the overall constrained goal of a realistic rendering. Genetic programs are functions that want to create wildly expansive shapes and patterns. But the shapes are being selected for how well they conform to the goal.
